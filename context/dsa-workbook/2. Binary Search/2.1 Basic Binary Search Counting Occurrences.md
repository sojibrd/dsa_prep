### 2.1 Basic Binary Search ও Counting Occurrences

**চিনবেন কীভাবে:** sorted array, "find efficiently", "first/last occurrence", "count of X in range"

**Demo: Find First and Last Position of Element** — [LC 34](https://leetcode.com/problems/find-first-and-last-position-of-element-in-sorted-array/) _(Medium — Google/Meta/Apple High)_

**Statement (Demo):** sorted integer array `nums` ও integer `target` দেওয়া — `target`-এর starting ও ending position `[first, last]` হিসেবে return করুন। না থাকলে `[-1, -1]`। O(log n) time-এ করতে হবে।
উদাহরণ: `nums=[5,7,7,8,8,10], target=8` → `[3,4]` | ⚡ `n ≤ 10⁵`, `-10⁹ ≤ nums[i] ≤ 10⁹`

**Approach:** দুইবার binary search — একবার left-biased (match পেলে বামে চাপুন), একবার right-biased (match পেলে ডানে চাপুন)। এটাই "counting occurrences"-এরও ভিত্তি: `last - first + 1`।

```js
function searchRange(nums, target) {
  const bound = (isFirst) => {
    let lo = 0,
      hi = nums.length - 1,
      ans = -1;
    while (lo <= hi) {
      const mid = (lo + hi) >> 1;
      if (nums[mid] === target) {
        ans = mid;
        if (isFirst)
          hi = mid - 1; // আরও বামে খুঁজুন
        else lo = mid + 1; // আরও ডানে খুঁজুন
      } else if (nums[mid] < target) lo = mid + 1;
      else hi = mid - 1;
    }
    return ans;
  };
  return [bound(true), bound(false)];
}
```

**Complexity:** Time O(log n), Space O(1)

      → Statement: sorted unique integer array `nums` ও integer `target` দেওয়া — `target` থাকলে তার index, না থাকলে in-order insert করলে যে index হবে সেটা return করুন। O(log n) time।
      উদাহরণ: `nums=[1,3,5,6], target=5` → `2` | ⚡ `n ≤ 10⁴`, `-10⁴ ≤ nums[i], target ≤ 10⁴`
      → আমার সমাধান: standard binary search. Target না পাওয়া গেলে লুপের শেষে `low` সূচকটিই হবে সঠিক সন্নিবেশ অবস্থান (insert position)।
      → যে সমস্যা হয়েছিল: array-তে target না থাকলে লুপ শেষে `high` নয়, `low` এর মান ট্র্যাকিং করা প্রয়োজন।
- [x] **Binary Search** — [LC 704](https://leetcode.com/problems/binary-search/) — ⚪ Bonus _(টেমপ্লেট মুখস্থ লেখার প্র্যাকটিস)_
      → Statement: ascending order-এ sorted integer array `nums` ও integer `target` দেওয়া — `target`-এর index return করুন। না থাকলে `-1`। O(log n) time।
      উদাহরণ: `nums=[-1,0,3,5,9,12], target=9` → `4` | ⚡ `n ≤ 10⁴`, `-10⁴ ≤ nums[i], target ≤ 10⁴`
      → আমার সমাধান: `low = 0`, `high = n - 1` এবং `mid = low + Math.floor((high - low) / 2)` ব্যবহার করে ইনডেক্স overflow এড়ানো।
      → যে সমস্যা হয়েছিল: লুপ কন্ডিশনে `lo <= hi` নিশ্চিত করা এবং integer division ঠিক রাখা।
- [x] **Median of Two Sorted Arrays** — [LC 4](https://leetcode.com/problems/median-of-two-sorted-arrays/) — 🔥 Must-do _(Hard — Google/Apple High; ছোট array-তে partition search)_
      → Statement: দুটো sorted array `nums1` (size m) ও `nums2` (size n) দেওয়া — দুটো merge করলে যে sorted array হবে তার median বের করুন। O(log(m+n)) time।
      উদাহরণ: `nums1=[1,3], nums2=[2]` → `2.0` | ⚡ `m,n ≤ 1000`, `-10⁶ ≤ nums[i] ≤ 10⁶`
      → আমার সমাধান: ছোট array-টির ওপর binary search চালিয়ে দুইভাগে ভাগ করা যাতে বাম পাশের max উপাদান ডান পাশের min উপাদানের চেয়ে ছোট বা সমান হয়।
      → যে সমস্যা হয়েছিল: index bounds হ্যান্ডেল করতে infinity ব্যবহার করা। ছোট array-তে বাইনারি সার্চ না করলে O(log(min(m, n))) পাওয়া যায় না।
